# 01_repo_map

This stage is for read-only codebase understanding.

The mini project is intentionally small, but it has enough structure to practice a real Codex habit: ask for understanding before asking for edits.

## Goal

Use Codex to answer questions like:

- What are the main modules?
- Where does the sample data come from?
- Which functions feel like entry points?
- What tests already describe behavior?

## What to practice

- Ask Codex to inspect files before suggesting changes
- Ask for a repo map or flow summary
- Avoid editing anything in this stage unless you deliberately want extra practice

## Run the tests

```powershell
python -m unittest discover -s tests
```
